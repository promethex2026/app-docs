## Services

### Frontend

| | |
|---|---|
| **Tech** | React 18, TypeScript, Vite, UnoCSS |
| **Hosting** | Cloudflare Workers (edge) |
| **Auth** | Privy (email, social, wallet) |
| **Wallet** | ERC-4337 Light Account via Alchemy |
| **State** | Zustand + TanStack React Query |

The frontend is a single-page app deployed to Cloudflare's edge network. It communicates with the Market Backend via REST API and receives real-time updates via SSE.

### Market Backend (BFF)

| | |
|---|---|
| **Tech** | Go, Kratos framework |
| **Port** | 8000 (HTTP) |
| **Role** | API gateway, auth, rate limiting, request routing |

The BFF (Backend for Frontend) is the only service exposed to the internet. It:
- Validates Privy JWT tokens
- Applies rate limiting (BBR adaptive algorithm)
- Routes requests to Market Service via gRPC
- Manages SSE connections for real-time events

### Market Service

| | |
|---|---|
| **Tech** | Go, Kratos framework |
| **Port** | 9000 (gRPC) |
| **Role** | Core business logic |

Two gRPC services:
- **Usercenter** — User management, social features, notifications
- **Marketcenter** — Markets, trading, positions, on-chain event processing

Handles order execution by submitting ERC-4337 UserOperations to the Alchemy bundler.

### Block Listener

| | |
|---|---|
| **Tech** | Go, daemon |
| **Role** | On-chain event indexer |

Continuously scans Arbitrum blocks for:
- Deposit/withdraw events (trades)
- Claim events (settlement payouts)
- Market status changes
- Dispute and resolution events

Processes events and updates the database via Market Service gRPC calls.

### Recommendation Service

| | |
|---|---|
| **Tech** | Python, gRPC |
| **Port** | 50051 |
| **Role** | ML-powered search and trending |

- Generates vector embeddings for semantic market search (pgvector)
- Computes trending/hot market rankings

### Admin Dashboard

| | |
|---|---|
| **Tech** | Go, Fiber framework |
| **Role** | Internal ops, market management |

Full admin panel for:
- Creating and managing prediction markets
- Publishing, settling, and canceling markets
- Content management (banners, categories, sections)
- User management with RBAC
- Multi-tenant support for white-label partners
- Transaction queue monitoring

Accesses PostgreSQL directly (not via Market Service gRPC).

## Data Stores

| Store | Purpose |
|-------|---------|
| **PostgreSQL** | Markets, orders, users, positions, transactions, content |
| **pgvector** | Semantic search embeddings (via Recommendation service) |
| **Redis** | Caching, distributed locks, rate limiting, block tracking |
| **S3** | Image and file storage |
| **Kafka** | Event bus for CLOB match engine (planned) |
