## Step by Step

### 1. Connect Wallet

<Steps>
  <Step title="Sign in via Privy">
    Choose email, Google, Twitter, or connect an external wallet (MetaMask, Coinbase Wallet).
  </Step>
  <Step title="Smart account created">
    A Light Account (ERC-4337) is automatically created. This is the user's on-chain identity â€” no seed phrase needed for social logins.
  </Step>
  <Step title="Backend registration">
    Frontend calls `POST /user/login` with the Privy JWT. Backend creates or retrieves the user record.
  </Step>
</Steps>

### 2. Discover Markets

Users browse markets on the home page:

- **Trending** â€” High-volume markets (`GET /market/get-hot-markets`)
- **By category** â€” Politics, Sports, Crypto, Culture
- **Search** â€” Keyword search across markets and users (`GET /base/search`)
- **Featured** â€” Curated banners and sections (`GET /market/get-banners`, `GET /market/get-sections`)

Each market card shows the title, current prices, volume, and time remaining.

### 3. Place a Trade

<Steps>
  <Step title="Select outcome">
    User picks an option (e.g., "Yes" at $0.65) and enters the amount in USDC.
  </Step>
  <Step title="Build UserOperation">
    Frontend encodes the contract call (`deposit` for buy, `withdraw` for sell) and creates an ERC-4337 UserOperation.
  </Step>
  <Step title="Gas sponsorship">
    Frontend requests paymaster data from `POST /market/get-paymaster-data`. The Alchemy paymaster sponsors all gas costs.
  </Step>
  <Step title="User signs">
    User signs the UserOperation with their embedded or external wallet. This is the only user action â€” one signature.
  </Step>
  <Step title="Submit">
    Frontend sends the signed UserOperation to `POST /market/place-order`. Backend submits it to the bundler.
  </Step>
  <Step title="Confirmation">
    Backend returns an `opHash`. The transaction is processed on Arbitrum within seconds. SSE pushes `user:position_changed` when complete.
  </Step>
</Steps>

| Portfolio value | `GET /market/get-user-asset-info` | SSE: `user:asset_changed` |
| Trade history | `GET /market/get-user-trades` | â€” |

### 5. Market Settlement

When the event outcome is known:

1. **Assertion** â€” Market owner asserts the result (e.g., "Yes won")
2. **Challenge window** â€” Configurable period where anyone can dispute
3. **Resolution** â€” If undisputed, the oracle confirms. If disputed, goes to arbitration.
4. **Status update** â€” Block Listener detects the on-chain event, backend updates market to "Ended"

### 6. Claim Winnings

<Steps>
  <Step title="Check eligibility">
    Users with winning option tokens see a "Claim" button on the market page.
  </Step>
  <Step title="Claim transaction">
    Same UserOperation flow: encode `claimAll()` â†’ paymaster sponsors gas â†’ user signs â†’ submit via `POST /market/claim-market-result`.
  </Step>
  <Step title="Receive USDC">
    Winning tokens are burned and USDC is transferred to the user's smart account. Losing tokens become worthless.
  </Step>
</Steps>

## Account Abstraction (ERC-4337)

Every on-chain action follows this pattern:

```
User signs UserOperation
    â†’ Backend adds paymaster data (Alchemy)
    â†’ Bundler submits to EntryPoint
    â†’ Paymaster sponsors gas
    â†’ Transaction executes on Arbitrum
```

<Info>

</Info>
