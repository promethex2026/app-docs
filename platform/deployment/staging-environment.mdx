## Quick Start

<Steps>
  <Step title="Open the staging app">
    Navigate to [staging.promethex.market](https://staging.promethex.market) in your browser.
  </Step>
  <Step title="Sign in">
    Click **Connect** and sign in with email, Google, Twitter, or an external wallet. Privy creates a smart account for you automatically — no wallet setup needed.
  </Step>
  <Step title="Get test USDC">
    Use the in-app faucet to receive tUSDC, or mint directly from the [tUSDC contract](0x1A8da4723A4f7aD0A965cb3dBC31c72435b1B490). Test tokens have no real value.
  </Step>
  <Step title="Browse markets">
    Explore active prediction markets. Each market shows current prices (probabilities), volume, and expiry date.
  </Step>
  <Step title="Place a trade">
    Select an outcome, enter your amount, and confirm. The trade executes on-chain with zero gas fees — the Paymaster covers everything.
  </Step>
  <Step title="Monitor positions">
    Track your open positions, P&L, and trade history in the portfolio view.
  </Step>
</Steps>

## API Access

The staging API is open for integration testing. All endpoints documented in the [API Reference](/api-reference/overview) are available.

```bash
# List active markets
curl https://staging.promethex.market/api/v1/market/get-markets?status=running

# Get market detail
curl https://staging.promethex.market/api/v1/market/get-market-detail?marketAddress=0x...

# Subscribe to real-time events (SSE)
curl -N https://staging.promethex.market/api/v1/sse/subscribe?channels=market:0x...
```

<Info>
  The staging environment uses the same API endpoints, authentication flow, and smart contracts as production. Code written against staging will work on mainnet with only a URL change.
</Info>

The staging environment runs on dedicated infrastructure that mirrors the production topology:

| **Storage**          | MinIO (S3-compatible) | Images, assets                                                                                              |

## Limitations

The staging environment has a few differences from production:

| Area        | Staging                    | Production             |
| ----------- | -------------------------- | ---------------------- |
| **Network** | Arbitrum Sepolia           | Arbitrum One           |
| **Token**   | tUSDC (test, mintable)     | USDC (real)            |
| **Oracle**  | MockOptimisticOracleV3     | UMA OptimisticOracleV3 |
| **Data**    | Test markets, may be reset | Persistent             |
| **SLA**     | Best effort                | Production SLA         |

<Note>
  The staging environment may be reset periodically as we deploy new features. Test data (markets, positions, trades) may not persist across resets.
</Note>
