## Three Ways to Integrate

PrometheX offers three integration models, each designed for different levels of customization and engineering effort. Pick the one that fits your product and team.

<CardGroup cols={3}>
  <Card title="White-Label" icon="store" href="/platform/integration/white-label">
    **Fastest to launch.** A fully branded prediction market experience deployed on your domain. No frontend code required.
  </Card>
  <Card title="Embedded" icon="window-restore" href="/platform/integration/embedded">
    **Drop-in component.** Add a prediction market widget to your existing app via iframe or React SDK.
  </Card>
  <Card title="API-Only" icon="code" href="/platform/integration/api-only">
    **Maximum control.** Build your own UI and UX from scratch using the PrometheX REST API and smart contracts.
  </Card>
</CardGroup>

| **Admin dashboard** | ✅ Full access | ✅ Full access | ✅ Full access |

| **Ideal for** | Fast launch, non-technical teams | Adding markets to existing app | Unique UX requirements |

## Decision Guide

<Accordion title="I want to launch as fast as possible">
**Choose White-Label.** You get a production-ready prediction market under your brand in 1–2 weeks. We configure your tenant (domain, theme, features, fees), and you're live. No frontend code, no backend work.

Best for: Crypto platforms, media companies, and communities that want to ship prediction markets without building anything.
</Accordion>

<Accordion title="I have an existing app and want to add prediction markets">
**Choose Embedded.** Drop a market widget into your current product with a few lines of code. Users can browse and trade prediction markets without leaving your app.

Best for: News apps, social platforms, and gaming sites that want to enrich their product with prediction markets as a feature.
</Accordion>

<Accordion title="I need complete control over the UX">
**Choose API-Only.** Use the PrometheX REST API to fetch markets, execute trades, and manage positions — then build whatever UI you want. You control every pixel.

Best for: Trading platforms, DeFi protocols, and teams with strong frontend engineering who want a bespoke experience.
</Accordion>

<Accordion title="I'm not sure yet — can I start small and upgrade?">
**Start with Embedded, upgrade to White-Label or API-Only later.** The underlying API and smart contracts are the same across all models. You can embed a widget today and migrate to a full white-label deployment when you're ready.
</Accordion>

## What's Shared Across All Models

Regardless of which integration model you choose, you get the same core infrastructure:

<CardGroup cols={2}>
  <Card title="Same smart contracts" icon="file-contract">
    All models use the same PredictionFactory, APMM engine, and resolution modules on Arbitrum.
  </Card>
  <Card title="Same API" icon="plug">
    REST API endpoints and SSE streams are identical. White-label and embedded just add a UI layer on top.
  </Card>
  <Card title="Same admin tools" icon="gauge-high">
    Every tenant gets full access to the Admin Dashboard for market management, content, and user oversight.
  </Card>
  <Card title="Same gasless UX" icon="gas-pump">
    ERC-4337 account abstraction and Paymaster are available across all integration models. Users never pay gas.
  </Card>
</CardGroup>
