## Overview

The Embedded model lets you add prediction markets to your existing application without building a new product. Drop in an iframe or use the React SDK to render a market widget — your users can browse and trade without leaving your site.

<Tip>
  Embedded integration takes **1–3 days** of engineering effort. You get a fully functional trading widget with a few lines of code.
</Tip>

## Integration Methods(Coming Soon)

<Tabs>
  <Tab title="iframe">
    The simplest approach. Embed a prediction market as an iframe that loads the PrometheX hosted widget.

    ```html
    <iframe
      src="https://embed.promethex.market/market/0x1234...abcd?theme=dark&tenant=your-tenant-id"
      width="100%"
      height="600"
      frameborder="0"
      allow="clipboard-write"
      style="border-radius: 12px; border: 1px solid #2a2a3e;"
    ></iframe>
    ```

    **Pros:** Zero dependencies, works in any web stack, minimal code. **Cons:** Limited cross-origin communication, styling constrained to config options.
  </Tab>
  <Tab title="JavaScript SDK">
    Load the PrometheX SDK script and initialize a market widget programmatically.

    ```html
    <script src="https://cdn.promethex.market/embed/v1/promethex.js"></script>
    
    <div id="promethex-widget"></div>
    
    <script>
      PrometheX.init({
        container: '#promethex-widget',
        apiKey: 'your-api-key',
        tenantId: 'your-tenant-id',
        marketAddress: '0x1234...abcd',
        theme: 'dark',
        onTradeComplete: function(trade) {
          console.log('Trade executed:', trade);
        }
      });
    </script>
    ```

    **Pros:** Event callbacks, dynamic configuration, programmatic control. **Cons:** Requires script tag, slight bundle size increase.
  </Tab>
  <Tab title="React SDK">
    For React applications, use the `@promethex/react` component library.

    ```bash
    npm install @promethex/react
    ```

    ```tsx
    import { PrometheXMarket, PrometheXProvider } from '@promethex/react';
    
    function App() {
      return (
        <PrometheXProvider
          apiKey="your-api-key"
          tenantId="your-tenant-id"
        >
          <PrometheXMarket
            marketAddress="0x1234...abcd"
            theme="dark"
            height={600}
            onTradeComplete={(trade) => {
              console.log('Trade executed:', trade);
            }}
          />
        </PrometheXProvider>
      );
    }
    ```

    **Pros:** Native React integration, TypeScript types, component lifecycle, event callbacks. **Cons:** React 18+ required.
  </Tab>
</Tabs>

## Configuration Options

| Option           | Type                  | Default      | Description                                  |
| ---------------- | --------------------- | ------------ | -------------------------------------------- |
| `marketAddress`  | `string`              | **required** | On-chain market contract address             |
| `tenantId`       | `string`              | **required** | Your tenant identifier                       |
| `apiKey`         | `string`              | **required** | API key for authentication                   |
| `theme`          | `"light"` \| `"dark"` | `"light"`    | Widget color theme                           |
| `width`          | `string`              | `"100%"`     | Widget width (px or %)                       |
| `height`         | `number`              | `600`        | Widget height in pixels                      |
| `locale`         | `string`              | `"en"`       | Language code (`en`, `zh`)                   |
| `hideHeader`     | `boolean`             | `false`      | Hide the market title bar                    |
| `tradingEnabled` | `boolean`             | `true`       | Enable trading (set `false` for view-only)   |
| `showChart`      | `boolean`             | `true`       | Show the price history chart                 |
| `showOrderBook`  | `boolean`             | `false`      | Show the order book (when CLOB is available) |

## Theming

Customize the widget appearance to match your application.

| Theme Property      | Type     | Description                           |
| ------------------- | -------- | ------------------------------------- |
| `colors.primary`    | `string` | Primary accent color (buttons, links) |
| `colors.background` | `string` | Widget background color               |
| `colors.text`       | `string` | Primary text color                    |
| `colors.success`    | `string` | Positive outcome / gain color         |
| `colors.danger`     | `string` | Negative outcome / loss color         |
| `borderRadius`      | `string` | Border radius for cards and buttons   |
| `fontFamily`        | `string` | Font family override                  |

```javascript
PrometheX.init({
  container: '#promethex-widget',
  apiKey: 'your-api-key',
  tenantId: 'your-tenant-id',
  marketAddress: '0x1234...abcd',
  theme: {
    mode: 'dark',
    colors: {
      primary: '#6366F1',
      background: '#0D0B1A',
      text: '#FFFFFF',
      success: '#22C55E',
      danger: '#EF4444'
    },
    borderRadius: '12px',
    fontFamily: 'Inter, system-ui, sans-serif'
  }
});
```

## Event Callbacks

Listen for user actions and state changes in the embedded widget.

| Event             | Payload                                                 | Description                         |
| ----------------- | ------------------------------------------------------- | ----------------------------------- |
| `onTradeComplete` | `{ marketAddress, optionIndex, amount, price, txHash }` | User completed a trade              |
| `onMarketLoad`    | `{ marketAddress, title, options, prices }`             | Market data loaded                  |
| `onPriceUpdate`   | `{ marketAddress, prices }`                             | Real-time price change              |
| `onError`         | `{ code, message }`                                     | Error occurred in widget            |
| `onAuthRequired`  | `{}`                                                    | User needs to authenticate to trade |

## Security

- All widget communication uses HTTPS
- API keys are scoped to your tenant with configurable permissions
- Trading requires user authentication via Privy (managed within the widget)
- Cross-origin isolation prevents data leakage between tenants
- Rate limiting protects against abuse
