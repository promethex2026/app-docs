## Overview

`Prediction` is the core market contract. Each instance is an [EIP-1167 minimal proxy](https://eips.ethereum.org/EIPS/eip-1167) clone deployed by [PredictionFactory](/contracts/prediction-factory). It implements the **Automated Prediction Market Maker (APMM)** — a weighted constant-product invariant adapted for multi-outcome prediction markets.

The contract itself is an **ERC-20 LP token**: liquidity providers receive `Prediction` tokens representing their share of the pool.

### Inheritance

    ERC20Upgradeable,                                // LP token

## State

### Core Storage

```solidity
struct MarketState {
    SD59x18[] weights;        // Price weights per option (sum = 1e18)
    SD59x18[] reserves;       // APMM reserves per option
    Option[] options;         // ERC-20 option token addresses
    uint256 fee;              // Trading fee in basis points (e.g., 200 = 2%)
    uint256 rewardPerShare;   // Accumulated fee reward per LP share
    Status status;            // Current market status
    uint256 settledOption;    // Index of the winning option (set after settlement)
}

enum Status {
    Created,    // Market deployed, not yet active
    Running,    // Trading active
    Settling,   // Outcome asserted, liveness period active
    Disputed,   // Assertion disputed, escalated to DVM
    Ended       // Resolved — claims available
}
```

Called by the factory immediately after clone deployment. Cannot be called again.

### Trading

#### `deposit` — Buy Option Tokens

```solidity
/// @notice Buy option tokens by depositing base tokens
/// @param optionIndex  Index of the option to purchase
/// @param amount       Base token amount to spend (before fees)
/// @param minReceive   Minimum option tokens to receive (slippage protection)
/// @return received    Actual option tokens received
/// @dev Reverts if market is not Running or received < minReceive
function deposit(
    uint256 optionIndex,
    uint256 amount,
    uint256 minReceive
) external nonReentrant returns (uint256 received)
```

**APMM calculation:**

1. Deduct fee: `netAmount = amount * (10000 - fee) / 10000`
2. Add `netAmount` to all reserves (increases liquidity)
3. Compute new reserve for `optionIndex` using the invariant
4. Difference = option tokens minted to caller

<Accordion title="Deposit math walkthrough">
Given current reserves `[r₀, r₁, …, rₙ]`, weights `[w₀, w₁, …, wₙ]`, and invariant `k`:

```
∏ rᵢʷⁱ = k
```

After depositing `netAmount` into all reserves:

```
r'ⱼ = rⱼ + netAmount   ∀ j ≠ optionIndex
```

Solve for the new target reserve:

```
r'_opt = (k / ∏(j≠opt) r'ⱼʷʲ) ^ (1/w_opt)
```

Tokens minted to user:

```
received = (r_opt + netAmount) - r'_opt
```
</Accordion>

#### `withdraw` — Sell Option Tokens

```solidity
/// @notice Sell option tokens to receive base tokens
/// @param optionIndex  Index of the option to sell
/// @param amount       Option token amount to sell
/// @param minReceive   Minimum base tokens to receive (slippage protection)
/// @return received    Actual base tokens received
function withdraw(
    uint256 optionIndex,
    uint256 amount,
    uint256 minReceive
) external nonReentrant returns (uint256 received)
```

Inverse of `deposit`: burns option tokens, removes proportional base tokens from reserves.

### Liquidity

#### `addLiquidity`

```solidity
/// @notice Provide liquidity and receive LP tokens
/// @param amount  Base token amount to deposit
/// @return lpTokens  LP tokens minted (proportional to pool share)
/// @dev Increases all reserves proportionally to maintain current prices
function addLiquidity(uint256 amount) external nonReentrant returns (uint256 lpTokens)
```

<Note>
Adding liquidity **does not** change option prices. Reserves scale proportionally, preserving the APMM invariant ratio.
</Note>

#### `removeLiquidity`

```solidity
/// @notice Burn LP tokens to withdraw base tokens + accumulated fees
/// @param lpAmount  LP tokens to burn
/// @return baseReturned  Base tokens returned (principal + fees)
function removeLiquidity(uint256 lpAmount) external nonReentrant returns (uint256 baseReturned)
```

LP tokens are burned, and the caller receives their proportional share of reserves plus any accumulated trading fees (tracked via `rewardPerShare`).

#### `settle`

```solidity
/// @notice Callback from UMA oracle when assertion resolves without dispute

/// @dev Market transitions Settling → Ended (if true) or back to Running (if false)
function assertionResolvedCallback(
    bytes32 assertionId,
    bool assertedTruthfully
) external
```

#### `assertionDisputedCallback`

```solidity
/// @notice Callback from UMA oracle when assertion is disputed
/// @param assertionId  The disputed assertion identifier
/// @dev Market transitions Settling → Disputed
function assertionDisputedCallback(bytes32 assertionId) external
```

#### `claimAll`

```solidity
/// @notice Redeem all winning option tokens for base tokens at 1:1
/// @dev Only callable when market status is Ended.
///      Burns all winning option tokens held by msg.sender.
///      Transfers equivalent base tokens.
function claimAll() external nonReentrant
```

## APMM Pricing

### Invariant

The APMM uses a **weighted constant-product** formula generalized from Uniswap's `x · y = k`:

```
∏ rᵢʷⁱ = k
```

Where:
- `rᵢ` = reserve of option `i`
- `wᵢ` = weight of option `i` (with `∑wᵢ = 1`)
- `k` = invariant constant (set at initialization)

### Price Formula

The instantaneous price of option `i` is:

```
Priceᵢ = wᵢ · (∑ rⱼ) / rᵢ
```

**Properties:**
- All prices sum to approximately 1 (ignoring fees)
- Price increases as demand (buying) reduces the option's reserve
- Price decreases as selling adds to the option's reserve

### Binary Market Example

For a Yes/No market with equal weights (`w = 0.5`):

```
Price_Yes = r_No / (r_Yes + r_No)
```

If reserves are `[800, 1200]` (Yes, No):

```
Price_Yes = 1200 / (800 + 1200) = 0.60  (60%)
Price_No  = 800  / (800 + 1200) = 0.40  (40%)
```

### Fee Mechanics

Trading fees are deducted before the APMM calculation:

```
netAmount = amount * (10000 - feeBps) / 10000
feeAmount = amount - netAmount
```

Fees accumulate in `rewardPerShare` and are distributed pro-rata to LP token holders upon `removeLiquidity`.

<CodeGroup>

```typescript viem
import { parseUnits } from "viem";
import { predictionAbi } from "./abis/Prediction";

const MARKET = "0x..."; // Prediction clone address
const TUSDC = "0x934E616cA9538397E3f9a12895C40ea2E8C45E48";

  abi: erc20Abi,

  address: MARKET,
  abi: predictionAbi,
  functionName: "quoteDeposit",
  args: [0n, parseUnits("100", 6)],
});

const minReceive = (quote * 99n) / 100n; // 1% slippage

const hash = await walletClient.writeContract({
  address: MARKET,
  abi: predictionAbi,
  functionName: "deposit",
  args: [0n, parseUnits("100", 6), minReceive],
});
```

```typescript ethers.js
import { ethers } from "ethers";

const market = new ethers.Contract(MARKET, predictionAbi, signer);
const usdc = new ethers.Contract(TUSDC, erc20Abi, signer);

await usdc.approve(MARKET, ethers.parseUnits("100", 6));

// Quote and buy with 1% slippage

await tx.wait();
```

## Security Considerations

- **Reentrancy**: All state-changing functions use `nonReentrant` modifier
- **Slippage**: `minReceive` parameters protect users from sandwich attacks
- **Ownership**: Two-step ownership transfer (`Ownable2StepUpgradeable`) prevents accidental transfers
- **Settlement**: Only the market owner can initiate settlement, but the oracle provides decentralized verification
- **Integer overflow**: All arithmetic uses Solidity 0.8+ checked math or PRBMath `SD59x18`
