## Overview

PrometheX uses a **modular resolution system**. Markets can be configured with different resolution modules depending on the use case.

## IResolutionModule Interface

```solidity
interface IResolutionModule {
    function assertOutcome(
        address market,
        uint256 optionIndex,
        bytes calldata data
    ) external returns (bytes32 assertionId);

    function resolveAssertion(bytes32 assertionId) external;

    function isResolved(bytes32 assertionId) external view returns (bool);
    function getOutcome(bytes32 assertionId) external view returns (uint256);
}
```

Any contract implementing this interface can serve as a resolution module.

## Available Modules

### UMA Optimistic Oracle (Default)

Uses UMA's OptimisticOracleV3 for decentralized dispute resolution.

**Flow:**

```
Owner asserts outcome
    → Bond posted to UMA
    → Liveness period (configurable)
        → No dispute → Outcome accepted
        → Disputed → Counter-bond posted
            → UMA DVM vote → Final resolution
```

**Configuration:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| Bond | Minimum required by UMA | Assertion bond amount |
| Liveness | 7200 seconds (2 hours) | Challenge window duration |
| Currency | USDC | Bond token |

**When to use:** Public markets where decentralized verification matters.

### MultisigResolution (Building)

N-of-M multisig for trusted resolver groups.

**Flow:**

```
Resolvers submit votes
    → Threshold reached (e.g., 3 of 5)
    → Outcome confirmed
    → Market settles
```

**When to use:** Internal markets, fast resolution needs, white-label deployments where the operator is trusted.

### Token Governance (Planned)

Token-weighted voting for market resolution.

**When to use:** High-stakes or controversial markets where broad community consensus is desired.

## Dispute Mechanics

### How disputes work (UMA)

1. **Assertion** — Market owner asserts outcome, bonds $X
2. **Challenge** — Disputer posts counter-bond of $X
3. **Escalation** — Goes to UMA Data Verification Mechanism (DVM)
4. **Voting** — UMA token holders vote on the correct outcome
5. **Resolution** — Winner receives loser's bond (minus oracle fee)

### Bond economics

| Role | Outcome | Gets |
|------|---------|------|
| Asserter (correct) | Undisputed | Bond returned |
| Asserter (correct) | Disputed, wins DVM | Bond + disputer's bond - fee |
| Asserter (wrong) | Disputed, loses DVM | Loses bond |
| Disputer (correct) | Wins DVM | Bond + asserter's bond - fee |
| Disputer (wrong) | Loses DVM | Loses bond |

The bond mechanism ensures economic incentive for honest assertion and honest disputes.
