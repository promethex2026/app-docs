## Current Fallback Mechanism

Manual fallback resolution is available through CTFOracle's **owner emergency mechanism**. When `manualFallbackEnabled` is set to `true`, the CTFOracle owner can directly resolve a market by calling:

```solidity
/// @notice Manual resolution fallback (owner only, requires manualFallbackEnabled)
/// @param questionId  The question to resolve
/// @param payouts     Payout array (e.g., [1, 0] for outcome 0 wins)
function resolve(bytes32 questionId, uint256[] calldata payouts) external onlyOwner
```

This bypasses the resolution module entirely and calls `ConditionalTokens.reportPayouts()` directly.

<Warning>
Manual resolution is intended for **emergency use only** — for example, if the UMA oracle is unresponsive or if a market needs to be voided urgently. In production, resolution should always flow through a registered `IResolutionModule`.
</Warning>

## Planned MultisigResolution Module

The planned module will support:

- **N-of-M trusted resolver voting** for market resolution
- **Fast resolution** without oracle liveness periods
- **White-label deployments** where decentralized dispute resolution is unnecessary

Check back for updates as the module progresses through development and audit.
