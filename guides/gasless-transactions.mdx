## Overview

Every transaction on PrometheX is **gasless** for the user. We use ERC-4337 Account Abstraction with Alchemy's infrastructure to sponsor all gas costs.

## How It Works

```
User action (e.g., buy "Yes")
    ↓
Frontend encodes contract call
    ↓
Creates UserOperation
    ↓
Requests paymaster data from backend
    ↓
User signs UserOperation
    ↓
Backend submits to Alchemy Bundler
    ↓
Bundler → EntryPoint → Paymaster pays gas
    ↓
Transaction executes on Arbitrum
    ↓
User pays $0
```

## UserOperation Structure

Every on-chain action is wrapped in a `UserOperation`:

```typescript
interface UserOperation {
  sender: string;              // Smart account address
  nonce: string;               // Account nonce (hex)
  initCode: string;            // "0x" (account already deployed)
  callData: string;            // Encoded contract call
  callGasLimit: string;        // Gas limit for execution
  verificationGasLimit: string; // Gas for validation
  preVerificationGas: string;  // Overhead gas
  maxFeePerGas: string;        // Max gas price
  maxPriorityFeePerGas: string; // Priority fee
  paymasterAndData: string;    // Paymaster sponsorship data
  signature: string;           // User's signature
}
```

## Step by Step

### 1. Encode the Contract Call

```typescript
import { encodeFunctionData } from 'viem';

const callData = encodeFunctionData({
  abi: predictionAbi,
  functionName: 'deposit',
  args: [optionIndex, amount, minReceive]
});
```

### 2. Get Paymaster Data

```bash
POST /bayes/market/get-paymaster-data
```

```json Request
{
  "userOperation": {
    "sender": "0x...",
    "nonce": "0x01",
    "callData": "0x...",
    "callGasLimit": "0x0",
    "verificationGasLimit": "0x0",
    "preVerificationGas": "0x0",
    "maxFeePerGas": "0x0",
    "maxPriorityFeePerGas": "0x0"
  }
}
```

```json Response
{
  "code": 0,
  "data": {
    "preVerificationGas": "0x...",
    "verificationGasLimit": "0x...",
    "callGasLimit": "0x...",
    "paymaster": "0x...",
    "paymasterData": "0x...",
    "paymasterVerificationGasLimit": "0x..."
  }
}
```

### 3. Sign the UserOperation

```typescript
const signature = await smartAccount.signUserOperation(userOp);
```

### 4. Submit the Order

```bash
POST /bayes/market/place-order
```

Include the complete signed UserOperation in the request body. The backend submits it to the Alchemy bundler.

## Smart Accounts

Each user gets a **Light Account** (ERC-4337 smart contract wallet):

- Created automatically on first login
- No seed phrase needed for social logins
- Compatible with external wallets (MetaMask, etc.)
- Supports batched transactions
- Upgradeable

<Info>
Smart accounts are counterfactual — the account contract is only deployed on the user's first transaction. The address is deterministic, so it's known before deployment.
</Info>
