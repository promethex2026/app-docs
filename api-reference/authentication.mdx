## Overview

PrometheX uses [**Privy**](https://privy.io) for authentication. Users log in via email, social accounts, or crypto wallets. Privy issues a JWT that authenticates all subsequent API requests.

## Auth Flow

<Steps>
  <Step title="User authenticates via Privy SDK">
    Email, social login, or wallet connection. Privy handles MFA and account linking.
  </Step>
  <Step title="Privy issues a JWT">
    Signed with ES256. Contains the user's Privy DID as the `sub` claim.
  </Step>
  <Step title="Send JWT with API requests">
    Include the token in the `Authorization: Bearer` header.
  </Step>
  <Step title="PrometheX validates the token">
    Checks audience, issuer, expiry, and signature against Privy's public keys.
  </Step>
  <Step title="Privy DID maps to PrometheX UID">
    The `sub` claim (Privy DID) is mapped to an internal PrometheX user ID.
  </Step>
</Steps>

## Token Format

The Privy JWT is a standard JWT with these claims:

| Claim | Value             | Description                           |
| ----- | ----------------- | ------------------------------------- |
| `alg` | `ES256`           | Signing algorithm (ECDSA P-256)       |
| `iss` | `privy.io`        | Issuer                                |
| `aud` | Your Privy App ID | Audience — must match your app        |
| `sub` | `did:privy:...`   | User's Privy decentralized identifier |
| `exp` | Unix timestamp    | Token expiration                      |
| `iat` | Unix timestamp    | Token issued at                       |

## Using the Token

Include the JWT in every authenticated request:

<CodeGroup>

```bash cURL
curl -X POST "https://api-staging.promethex.market/api/v1/market/place-order" \
  -H "Authorization: Bearer eyJhbGciOiJFUzI1NiIs..." \
  -H "Content-Type: application/json" \
  -d '{ ... }'
```


```javascript JavaScript
const response = await fetch('https://api-staging.promethex.market/api/v1/market/place-order', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${privyToken}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ ... }),
});
```


```python Python
import requests

headers = {
    'Authorization': f'Bearer {privy_token}',
    'Content-Type': 'application/json',
}
response = requests.post(
    'https://api-staging.promethex.market/api/v1/market/place-order',
    headers=headers,
    json={ ... },
)
```

</CodeGroup>

## First-Time Login

When a user authenticates for the first time, register them with PrometheX:

```
POST /api/v1/user/login
```

<ParamField body="address" type="string" required>
  User's smart account address (ERC-4337 account).
</ParamField>

<ParamField body="inviteByCode" type="string">
  Referral code from another user.
</ParamField>

<ParamField body="userName" type="string">
  Display name.
</ParamField>

<ParamField body="avatarUrl" type="string">
  Avatar image URL.
</ParamField>

**Response:**

```json
{
  "code": 0,
  "data": {
    "uid": "12345",
    "isNewUser": true
  }
}
```

<ResponseField name="uid" type="string">
  PrometheX internal user ID.
</ResponseField>

<ResponseField name="isNewUser" type="boolean">
  `true` if the user was just created. `false` for returning users.
</ResponseField>

## Endpoint Auth Requirements

### Public (no auth)

| Endpoint                               | Description      |
| -------------------------------------- | ---------------- |
| `GET /api/v1/market/get-markets`       | List markets     |
| `GET /api/v1/market/get-market-detail` | Market detail    |
| `GET /api/v1/market/get-hot-markets`   | Trending markets |
| `GET /api/v1/market/get-price-history` | Price charts     |
| `GET /api/v1/market/get-market-trades` | Trade history    |

### Authenticated

| Endpoint                                  | Description        |
| ----------------------------------------- | ------------------ |
| `POST /api/v1/user/login`                 | Register / login   |
| `POST /api/v1/market/place-order`         | Buy or sell tokens |
| `POST /api/v1/market/claim-market-result` | Claim winnings     |
| `POST /api/v1/market/add-liquidity`       | Add liquidity      |
| `POST /api/v1/market/remove-liquidity`    | Remove liquidity   |
| `POST /api/v1/sse/connect`                | SSE stream         |
| `POST /api/v1/user/set-avatar`            | Update avatar      |
| `POST /api/v1/user/set-username`          | Update username    |
| `POST /api/v1/market/follow-market`       | Follow a market    |

<Info>
  Each endpoint's reference page indicates whether authentication is required.
</Info>

## Token Expiry & Refresh

Privy JWTs have a short lifespan. The Privy SDK handles token refresh automatically. If you receive a `401` or error code `110303` (PRIVY), the token may be expired — refresh it via the Privy SDK and retry.
