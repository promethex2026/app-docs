---
title: Place Order
description: "Buy or sell option tokens on a prediction market"
---

## Endpoint

```
POST /bayes/market/place-order
```

**Auth required:** Yes

## Request Body

```json
{
  "marketAddress": "0x1234...abcd",
  "optionAddress": "0xaaaa...1111",
  "side": 1,
  "price": "650000",
  "amount": "10000000",
  "minReceiveAmount": "9500000",
  "deadline": "1740000000",
  "userOperation": {
    "sender": "0x...",
    "nonce": "0x...",
    "initCode": "0x",
    "callData": "0x...",
    "callGasLimit": "0x...",
    "verificationGasLimit": "0x...",
    "preVerificationGas": "0x...",
    "maxFeePerGas": "0x...",
    "maxPriorityFeePerGas": "0x...",
    "paymasterAndData": "0x...",
    "signature": "0x..."
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `marketAddress` | string | Prediction market contract address |
| `optionAddress` | string | Option token to buy/sell |
| `side` | number | `1` = buy (deposit), `2` = sell (withdraw) |
| `price` | string | Price in base token units |
| `amount` | string | Amount in base token units |
| `minReceiveAmount` | string | Minimum output (slippage protection) |
| `deadline` | string | Unix timestamp deadline |
| `userOperation` | object | Signed ERC-4337 UserOperation |

## Response

```json
{
  "code": 0,
  "data": {
    "opHash": "0xabcdef...",
    "orderUuid": "order-123-456"
  }
}
```

The `opHash` can be used to track the transaction status. Once confirmed on-chain, an SSE event `user:position_changed` will be emitted.

<Warning>
The `userOperation` must be properly constructed and signed. See the [Gasless Transactions guide](/guides/gasless-transactions) for the full flow.
</Warning>
