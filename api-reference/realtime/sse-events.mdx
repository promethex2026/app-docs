## Overview

PrometheX uses **Server-Sent Events (SSE)** for real-time data. Connect once, receive push updates for prices, trades, and position changes.

## Connect

```
POST /bayes/sse/connect
```

**Auth required:** Yes

Returns an SSE stream. First event:

```
event: base:connected
data: {"connection_id": "conn-abc-123"}
```

Save the `connection_id` for subscribing to market topics.

### Example (JavaScript)

```javascript
const eventSource = new EventSource('/bayes/sse/connect', {
  headers: { 'Authorization': 'Bearer <token>' }
});

eventSource.addEventListener('base:connected', (e) => {
  const { connection_id } = JSON.parse(e.data);
  // Now subscribe to market topics
});

eventSource.addEventListener('market:price_change', (e) => {
  const data = JSON.parse(e.data);
  // Update UI with new prices
});
```

## Subscribe to Markets

```
POST /bayes/sse/subscribe
```

```json Request
{
  "connection_id": "conn-abc-123",
  "topics": [
    "market:0x1234...abcd",
    "market:0x5678...efgh"
  ]
}
```

## Unsubscribe

```
POST /bayes/sse/unsubscribe
```

Same format as subscribe.

## Event Types

### Automatic Events (no subscription needed)

| Event | Payload | Trigger |
|-------|---------|---------|
| `base:connected` | `{ connection_id }` | Connection established |
| `base:ping` | `{}` | Keepalive (every 30s) |
| `user:position_changed` | `{ uid, market_address, option_address }` | Trade executed |
| `user:asset_changed` | `{ uid, base_token_address }` | Balance updated |
| `user:new_notification` | `{ uid }` | New notification |

### Subscription Events (subscribe to specific markets)

| Event | Payload | Trigger |
|-------|---------|---------|
| `market:new_trades` | `{ market_address, option_address }` | New trade on market |
| `market:price_change` | `{ market_address, option_address }` | Price movement |
| `market:info_update` | `{ market_address, option_address }` | Market metadata change |

<Info>
The SSE connection has a 10-minute timeout. Reconnect when the connection drops. The `base:ping` event confirms the connection is alive.
</Info>
